Why do we bother with Inline elements?These are ceertainly times we want ot mark up content,but we don't want to insert the line breaks around them.The <span> tag was the firstto give you that ability,but it was mostly used to style the code,and we Don't want to do that.Some of the newer tags that still separate content and style are listed below.Remember,inline tags can flow next to each other and don't cause a line break before and after the element.We have seen this with links and images.

Citing a Source-Cite tag

The <cite> makes it much easier for people and bots to search the code for specific information.
Agreat refernce for these and other elements is https://developer.mozilla.org/en-US/docs/Web/HTML

Defining Your abbreviations -abbr tag

The <abbr> will display an abbrevaition on the browser.But if a title attribute is specified in the tag,the title will appear when the mouse hovers over the abbreviation.Screen readers also provide the title information.

The days of the weak are often abbreviated:Sun,Mon,Tue,etc.

Specifying the use of time-Time tag

The <time> tag defines a human-readable date/time.
This element can also be used to encode dates and times in a machine-readable way so that user agents can offer to add birthday reminders or scheduled events to the user's calendar, and search engines can produce smarter search results.

The party starts at 9:00P.M.

Identifying Code Fragments - code tag

The <code> tag is a phrase tag. It defines a piece of computer code.

C statement:- printf("Hello world");

The <code> element tends to be displayed in a different font,but also has the semantic meaning to convey information to as many people as possible.

Using Superscripts and Subscripts-Sub and Sup tag

The <sub> and <sup> elements are used to create the super and subscripts,e.g. (x+y)3 or H2O.
In the past people would often paste mathematical images into the page.But images don't convey the same meaning to all users.
